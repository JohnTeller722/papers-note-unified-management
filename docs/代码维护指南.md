# 代码维护指南（功能 -> 代码映射）

> 目标：你后续自己改功能时，能快速定位“改哪里”。

## 1. 快速定位总览

- 后端入口与路由：`backend/src/server.js`
- 数据库与数据访问：`backend/src/db.js`
- 前端主逻辑：`frontend/src/main.js`
- 前端接口封装：`frontend/src/api/client.js`
- 样式与布局：`frontend/src/styles.css`
- 页面骨架（侧边栏结构）：`frontend/index.html`
- 主题切换：`frontend/src/theme/index.js`

## 2. 按功能定位代码

## 2.0 AI 能力（字段生成 / 会话复用 / 批量分析）

- 前端录入页字段 AI 按钮与会话状态：`frontend/src/main.js`
  - `runFieldAiSuggest()`
  - `ensureAiFullPdfConsentIfNeeded()`
  - `resetAiSessionLocal()`
- 前端批量分析：`frontend/src/main.js`
  - `runBatchAnalyzeSelected()`
- 前端 API：`frontend/src/api/client.js`
  - `aiSuggest()`
  - `batchAnalyze()`
  - `getAiJob()`
  - `resetAiSession()`
- 后端 AI 单篇与批量：`backend/src/server.js`
  - `POST /api/ai/analyze-and-fill`
  - `POST /api/ai/batch/analyze`
  - `GET /api/ai/jobs/:id`
  - `POST /api/ai/session/reset`
- 后端 AI 配置与库表：`backend/src/db.js`
  - `app_settings` 扩展 AI 字段
  - `ai_jobs` 扩展状态字段
  - `ai_job_items`（批量逐条结果）
- PDF 全文提取：`backend/src/pdf.js`
  - `parsePdfFullText()`

**想改什么看哪里：**

- 改 AI 请求参数与模型调用：`backend/src/server.js` 的 `callOpenAiCompatible()` 与 AI 路由
- 改会话过期策略（当前 20 分钟）：`AI_SESSION_TTL_MS` in `backend/src/server.js`
- 改全文大小限制（当前 10MB）：`AI_MAX_PDF_BYTES` in `backend/src/server.js`
- 改前端确认弹窗与逐条采纳：`askConfirm()` 与 `applyFieldSuggestionWithConfirm()` in `frontend/src/main.js`

## 2.2 首页仪表盘（最近阅读/精读候选/速度盘）

- 首页渲染：`frontend/src/main.js` 的 `renderHome()`
- 数据计算：
  - 最近阅读：`getRecentPapers()`
  - 精读候选：`getDeepCandidates()`
  - 统计可视化：`gaugeMax/gaugePercent/gaugeDeg`
- 快捷交互：
  - 编辑：`jumpToEditPaper()`
  - 升级精读：`api.upgradePaper()`
  - 快捷导出：`exportByIds()` / `exportByFilters()`
- 样式：`frontend/src/styles.css`
  - `.gauge-card`, `.gauge`, `.gauge-needle`, `.gauge-stats`

**注意：**

- 速度盘当前是前端计算型展示，不依赖额外后端接口。
- 指针角度映射使用半圆区间（`-90° ~ 90°`），避免越界到下半区。

## 2.3 录入页（表单、保存弹窗、清空）

- 录入页渲染：`frontend/src/main.js` 的 `renderEntry()`
- 提交数据组装：`collectEntryPayload()`
- 保存后清空：`resetEntryAfterSave()`
- 成功/失败弹窗：`showModal()`
- 后端新建/更新：
  - `POST /api/papers` in `backend/src/server.js`
  - `PUT /api/papers/:id` in `backend/src/server.js`

**常见改动示例：**

- 新增基础字段：
  1) 前端 `BASE_ENTRY_FIELDS`
  2) 后端 `papers` 表字段（若要落主表）或 `fieldValues`（若走动态字段）
- 修改“保存成功”文案：`showModal` 调用处。

## 2.4 模板系统（多套粗读/精读）

- 前端模板管理 UI：`frontend/src/main.js` 的 `renderSettings()`（模板设计器区块）
- 前端模板选择：`renderEntry()` 中模板下拉
- API 封装：`frontend/src/api/client.js`
  - `getTemplates/createTemplate/updateTemplate/deleteTemplate`
- 后端路由：`backend/src/server.js`
  - `/api/templates` 系列
- 数据层：`backend/src/db.js`
  - `listTemplates`, `createTemplate`, `updateTemplate`, `deleteTemplate`, `getTemplateById`
- 默认模板字段：`backend/src/constants.js`

**注意：**

- 精读继承粗读靠 `template_defs.base_template_id`。
- 深模板字段由“继承字段 + 自身字段”合并得到（`getTemplateById`）。

## 2.5 枚举选项管理（类别/优先级/关系）

## 2.7 记录中心（筛选、编辑、升级、删除）

## 3. 数据库改动建议流程（自己改时尽量按这个做）

1. 先改 `backend/src/db.js` 的建表/迁移逻辑。
2. 再改 `server.js` 的读写路由。
3. 再改 `frontend/src/api/client.js`。
4. 最后改 `frontend/src/main.js` 的 UI 读写逻辑。
5. 本地启动后做一轮最小回归：
   - 录入保存
   - 记录筛选
   - 打开MD
   - 导出

---

## 4. 常见修改任务速查

- “我要新增一个筛选条件”
  - 前端：`state.filters` + `renderLibrary()` 筛选控件 + `api.listPapers` 参数
  - 后端：`listPapers(filters)` 增加 `WHERE`

- “我要新增模板字段类型（比如日期）”
  - 前端：`FIELD_TYPES`、`buildFieldInput()`
  - 后端：一般无需改表（动态值已支持），必要时改导出展示

- “我要加一个新页面”
  - `frontend/index.html` 增加 `section`
  - `renderTabs()` / `setActiveTab()` 增加对应分支
  - `styles.css` 增加页面样式

- “我要把 AI mock 换成真实模型”
  - 改 `POST /api/ai/analyze-and-fill` 的实现
  - 保留返回结构 `suggestedValues/confidence/rationale`，减少前端改动

---

## 5. 调试建议

- 优先看浏览器控制台（前端错误）与后端启动终端（接口错误）。
- 若提示端口占用，先清理旧 `node` 进程再启动。
- 若出现中文乱码，先区分：
  - 是“终端显示编码问题”还是“数据库实际脏数据”。
- 每次改完重点功能，至少走一遍真实点击路径，不只看单个接口。
