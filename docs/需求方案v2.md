# 文献笔记管理系统需求方案（V2）

## Context

当前痛点是“读完文献后记录不成体系、后续难复用”。本系统要把阅读过程标准化：先粗读、后精读，结构化沉淀文献信息，并支持导出复用。除基础记录外，还需支持 PDF 拖入后的元数据自动读取，以及后续大模型对已有数据进行综合分析并自动填充字段（人工可控）。

## 已确认关键决策

1. 主存储：`SQLite`（按需导出 `Excel`）。
2. 运行形态：本地 `Web` 单机版。
3. 模板策略：两套模板，但“精读模板 = 粗读模板的扩展与细化”。
4. MD 联动：记录本地路径，点击后本地直开。
5. PDF 解析：仅本地解析，不上传云端。
6. 粗读升级精读：同一条记录升级（`stage: rough -> deep`）。

## 范围与目标

### MVP（本期）

- 模板体系：粗读基础模板 + 精读扩展模板（继承关系）。
- 录入方式：手填 + 拖入 PDF 自动解析基础元数据（题目/作者/年份/来源信息）。
- 阅读流转：支持“粗读记录一键升级为精读记录”，继承已填字段并补充精读字段。
- 记录中心：检索、筛选、排序、批量操作。
- 导出：勾选导出、筛选结果导出、全量导出（Excel）。
- 选项管理：相关类别/优先级/和我关系等下拉值可新增、编辑、删除。
- 主题：日/夜间模式切换并记忆。
- MD 外挂：每条记录可绑定 md 路径并一键打开。

### 二期（预留，不在本期实现）

- 大模型综合分析：结合论文元数据、已填字段、历史记录自动补全更多字段。
- AI 批量处理队列：批量解析、批量建议填充、人工审核面板。
- 多设备同步/协作权限。

## 业务流程（用户视角）

1. 新建记录时选择粗读（默认）或直接精读。
2. 可先拖入 PDF 自动带出元数据，再补充其余字段。
3. 粗读完成后，若决定深入阅读，可“升级为精读”并继续补填。
4. 在记录中心进行筛选与批量导出。
5. 点击“打开 MD”跳转到本地笔记。

## 字段与模板设计

字段体系来源：`d:/002_codes/vibe_coding/文献阅读管理工具/模糊需求文档.md`。

### 粗读模板（Base）

- 题目、年份、会议/期刊、任务、方法、主要贡献、局限性、和我的关系、类别、精读优先级、链接。

### 精读模板（Extend Base）

- 自动包含粗读全部字段。
- 新增扩展字段：序号、作者、检索来源、命中关键词、方法关键词、数据集/实验场景、核心方法一句话、对比方法/基线、结果亮点、是否值得精读、GitHub/代码、我的备注。

### 可配置下拉项（非固定）

- 相关工作类别、精读优先级、和我的关系均支持：新增/编辑/删除/排序/启停用。
- 变更策略：选项删除不影响历史记录文本；仅影响后续可选项。

## 信息架构与页面

1. 首页：最近阅读、精读候选、导出快捷入口。
2. 录入页：模板选择、PDF 拖入解析、分区表单、保存。
3. 记录中心：表格视图（筛选/排序/批量导出/升级为精读）。
4. 模板与选项设置：模板字段开关、下拉项维护。
5. 系统设置：主题、导出目录、数据库备份、AI 配置（预留）。

## 数据模型草案（SQLite，按上述修改）

- `template_defs`：模板定义（`id`,`name`,`kind`,`base_template_id`,`schema_json`,`version`,`is_active`,`updated_at`）。
  - 说明：`base_template_id` 支持“精读继承粗读”。
- `template_fields`：模板字段元数据（`id`,`template_id`,`field_key`,`label`,`field_type`,`required`,`order_no`,`is_builtin`）。
- `enum_groups`：下拉组（`id`,`group_key`,`name`,`is_system`）。
- `enum_options`：下拉选项（`id`,`group_id`,`value`,`label`,`order_no`,`is_active`）。
- `papers`：文献主记录（`id`,`stage`,`title`,`year`,`venue`,`authors`,`link`,`pdf_path`,`md_path`,`created_at`,`updated_at`）。
  - 说明：`stage` 取值 `rough|deep`，支持状态升级。
- `paper_field_values`：字段值（`id`,`paper_id`,`field_key`,`value_text`,`value_json`,`source`）。
  - 说明：`source` 标记 `manual|pdf_auto|ai_suggested|ai_confirmed`。
- `paper_transitions`：阶段流转日志（`id`,`paper_id`,`from_stage`,`to_stage`,`operator`,`created_at`）。
  - 说明：默认同一 `paper_id` 完成粗读到精读升级，不复制新文献实体。
- `exports`：导出任务日志（`id`,`filter_json`,`selected_ids_json`,`file_path`,`status`,`created_at`）。
- `ai_jobs`（预留）：AI 分析任务（`id`,`paper_id`,`job_type`,`input_ref`,`status`,`result_json`,`created_at`）。

## PDF 元数据自动读取方案

- 触发：在录入页拖入 PDF。
- 解析边界：仅本地解析，不上传原文到云端服务。
- 输出：优先抽取题目、作者、年份、DOI/来源线索；自动回填到对应字段。
- 置信度：解析结果显示“自动填充”标识，可人工覆盖。
- 失败兜底：解析失败时不阻塞手填，并允许用户手动补全。

## AI 综合分析与自动填充（预留）

- 预留接口：`POST /api/ai/analyze-and-fill`。
- 入参：`paper_id`、当前字段值、可选原文片段/摘要。
- 出参：`suggested_values`、`confidence`、`rationale`。
- 策略：可按字段逐项接受；默认不直接覆盖人工内容。

## 导出策略

- 入口：记录中心（勾选/筛选结果/全量）。
- 格式：`.xlsx` 主格式，`.csv` 兜底。
- 列规则：可选“按粗读视图导出”或“按精读全字段导出”。

## MD 外挂方案

- 字段：`md_path` 本地绝对路径。
- 行为：点击后调用系统默认 md 编辑器打开。
- 异常：路径不存在时提示并支持重新绑定。

## 非功能与验收指标

- 可用性：10,000 条记录内检索/筛选响应 < 1 秒。
- 可靠性：录入草稿可恢复，异常退出不丢最近输入。
- 易用性：界面简洁、分区清晰、深浅色视觉一致。

## 验证方案（端到端）

1. 新建粗读记录：拖入 PDF 后自动带出元数据，手动补充并保存成功。
2. 将粗读记录升级为精读：原字段保留，新增字段可继续填写并保存。
3. 在设置页新增/修改/删除下拉选项，录入页选项即时生效。
4. 记录中心执行勾选导出与全量导出，检查列结构与数据完整性。
5. 切换日/夜模式并重启页面，主题偏好保持。
6. 绑定并打开本地 md 路径；非法路径提示正确。
7. 模拟 AI 建议填充返回，验证“建议可采纳、默认不覆盖人工值”。
